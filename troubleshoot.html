常见问题排错卡：按阶段速查

把各阶段最容易翻车的点集中在这里。先点上方筛选你所在的阶段，对照症状 → 原因 → 解法三步走。硬件世界里「看不见」，串口日志就是你的眼睛。

预备阶段 · 电路与 PCB

🛠️PCB 不通电 / 芯片发烫 / Wi‑Fi 弱
症状

板子回来不通电、芯片烫手、无线信号极弱、偶尔莫名复位。

原因

线宽太细载流不足；电源脚缺去耦电容；天线下方铺铜/走线屏蔽；封装画错焊不上；GND 没连成整片。

解法

电源/地加粗(≥0.5mm)；每 IC 旁放 0.1μF 陶瓷电容；天线区净空；打样前 ERC+DRC 清零；整板铺地铜保持低阻抗回路。

入门阶段

🛠️灯不亮
症状

代码跑起来了，LED 不闪。

原因

① LED 正负极接反；② 没串电阻或电阻太大；③ 引脚号写错；④ 板载灯是 GPIO2，外接要确认脚。

解法

LED 长脚接 GPIO、短脚接 GND；串 220Ω；确认脚号；先试板载蓝灯(GPIO2)排除接线问题。

🛠️端口找不到 / 烧录失败
症状

Thonny 找不到串口，或上传报错。

原因

没装 USB 驱动(CH340/CP210x)；板子没进下载模式；端口被占用；线只是充电线。

解法

装对应驱动；换数据线；设备管理器/ls /dev/cu.* 查端口；必要时长按 BOOT 再上传。

🛠️读按钮全 1 / 全 0（无变化）
症状

按按钮读到的电平不变，或一直 1。

原因

引脚浮空(没启用上拉/下拉)；按钮接法错；代码读成 OUTPUT。

解法

用 Pin(Pin.IN, Pin.PULL_UP)；一端接 GPIO、一端接 GND；按下读到 0、松开读到 1。

🛠️串口乱码
症状

串口打印一堆乱码。

原因

波特率不匹配(常见 115200)；编码问题；线接触不良。

解法

确认两端波特率一致；重插 USB；换线。查看 入门6 · 串口监视器。

中级阶段

🛠️ADC 读数跳变 / 不准
症状

电位器没动，读数上下乱跳；或范围不是 0~4095。

原因

没设 atten 导致上限只有 1V；电源纹波；浮空；噪声。

解法

用 adc.atten(machine.ADC.ATTN_11DB) 扩到 0~3.3V；软件平均多次采样；电源旁加去耦电容。

🛠️PWM 占空比报错
症状

duty() 报范围错或没效果。

原因

新旧版 API 不同：老 duty(0~1023)，新 duty_u16(0~65535)。

解法

print(pwm) 看提示；按版本换 API。详见 中级2 · PWM。

🛠️Wi‑Fi 连不上
症状

w.isconnected() 一直 False。

原因

只支持 2.4GHz（连 5GHz 必失败）；SSID/密码错；信号弱；路由器 MAC 过滤。

解法

确认 2.4GHz；核对密码；靠近路由器；串口打印状态排查。见 中级2。

🛠️WebServer 拒绝连接
症状

浏览器打不开 ESP32 的 IP。

原因

手机/电脑没和 ESP32 同 Wi‑Fi；地址填错；服务器死循环没 accept。

解法

同网段；用 http://<IP>；检查代码缩进确保持续 accept。见 中级3。

🛠️I2C 屏/传感器不亮或读不到
症状

OLED 黑屏，或 i2c.scan() 找不到设备。

原因

SDA/SCL 接反；地址不对；没上拉(模块常已带)；供电不足。

解法

确认 SDA=GPIO21、SCL=GPIO22；i2c.scan() 看真实地址；核对库初始化地址。见 中级5。

🛠️深度睡眠醒不来
症状

进深度睡眠后再也唤不醒。

原因

唤醒源(按钮/RTC)没配置；唤醒脚接法错；EXT0/EXT1 参数错。

解法

用 machine.wake_reason() 调试；确认唤醒脚接法与上拉；先试定时器唤醒。见 中级8。

高级阶段

🛠️uasyncio 卡死 / 只跑一个任务
症状

程序卡住，或只执行其中一个协程。

原因

协程里漏 await；用了阻塞 time.sleep；没 gather 并发；入口没 asyncio.run。

解法

一律用 await asyncio.sleep()；gather(a(),b()) 并发；asyncio.run(main()) 入口。见 高级1。

🛠️ESP‑NOW 收不到
症状

发送端发了，接收端 recv() 一直等不到。

原因

MAC 错/广播地址不对；信道不一致；双方没都 import espnow 并 active；发送端没 add_peer。

解法

双方 STA 都 active；peer 用正确 MAC 或广播；同信道；收发都 active 且已 add_peer。见 高级2。

🛠️MQTT 连不上 / 收不到
症状

connect() 失败或发布后订阅端收不到。

原因

broker 地址/端口错；板子未连 Wi‑Fi；订阅端没先 subscribe；主题大小写不一致。

解法

用公共 broker；确认 isconnected()；先 subscribe 再收；主题两端逐字一致。见 高级3。

🛠️BLE 手机搜不到
症状

nRF Connect 搜不到，或连上读不到数据。

原因

GATT 没注册成功；UUID 不符；用了经典蓝牙 App；iOS 缓存。

解法

确认 active 且 register 成功；用标准 128 位 UUID；BLE 专用 App；iOS 删设备重连。见 高级4。

🛠️电机不转 / 只响 / 单向
症状

电机不转、只嗡嗡响、或只能一个方向。

原因

驱动板没独立电源或没共地；IN1/IN2 逻辑错；电流超驱动板能力；缺续流二极管烧芯片。

解法

驱动板接独立电源并和 ESP32 共 GND；IN1/IN2 一高一低控方向；确认电流够；加续流二极管。见 高级5。

🛠️触摸/霍尔/红外 失效
症状

触摸读数不变；霍尔无反应；红外收不到。

原因

触摸脚用错(需 TouchPad 专用脚)；霍尔模拟/数字脚类型错；红外 OUT 接错或被强光干扰；阈值不合理。

解法

触摸用 GPIO32/33/27；按模拟/数字选对脚；红外 OUT 接数字 GPIO、远离日光灯；先打印原始值再定阈值。见 高级6。

🛠️OTA 变砖
症状

升级后起不来、反复重启或连不上。

原因

下载写了一半就重启；新代码有错；单文件覆盖无备份；没校验。

解法

升级前备份旧版；校验完整性再写；上 ESP‑IDF 用 A/B 双分区自动回滚。见 高级7。

专家阶段 · 常见卡点

🛠️ESP‑IDF 编译报错 / 组件找不到
症状

idf.py build 失败，提示找不到 component 或头文件。

原因

没 idf.py set-target esp32；组件没放进 components/；CMakeLists.txt 没加源文件；IDF 版本与例程不匹配。

解法

先 set-target；组件按规范放目录；CMake 注册源文件；用与教程一致的 IDF 版本（或 git submodule update）。见 专家1。

🛠️JTAG 连不上 / 调试器无响应
症状

OpenOCD 连不上芯片，或断点不生效。

原因

JTAG 引脚被复用(如 GPIO12/15 启动 strapping)；接线顺序错；目标配置错。

解法

核对 ESP32 JTAG 引脚(MTDI/TCK/TMS/TDO/TDI)；用 ESP‑Prog 或兼容调试器；先能正常 flash 再上 JTAG。见 专家2。
