认识 ESP32：你手里这块「带 Wi‑Fi 的小电脑」

时长 ≈ 1.5h · 入门阶段
🧍
心智图像：想象一个戴耳机（Wi‑Fi/蓝牙天线）的小工人，住在绿色电路板里。他有几十只「手」（GPIO 引脚）可以伸出去按开关、读传感器；他有一张「待办清单」（你写的程序）；他还能通过无线和你的手机/路由器聊天。他不需要屏幕、不需要键盘——你用一根 USB 线把清单喂给他就行。

ESP32 是一颗 系统级芯片（SoC）：CPU、无线、内存、各种外设都集成在一颗指甲盖大小的芯片里，外面再焊上一块 USB 转串口、一个电压稳压器、一根天线，就成了你买的「开发板」。

它强在哪

  • 双核 240MHz，比 Arduino UNO 快几十倍
  • 内置 Wi‑Fi + 蓝牙
  • ~30 个可用 GPIO
  • 支持 PWM / ADC / I2C / SPI / 触摸

和谁比

  • vs 树莓派：更小更便宜，无 Linux、无屏幕，但实时性更好
  • vs Arduino UNO：多了无线、快得多、引脚更多

代价

  • 内存小（几百 KB），跑不了大型程序
  • 没有操作系统界面
  • 电压是 3.3V，别接 5V 设备直接怼

一块开发板长这样（把鼠标移到引脚上看说明）

交互 · ESP32 引脚图悬停引脚查看功能
新手最常踩的 3 个坑 ① 把 5V 设备直接接到 3.3V 引脚 → 可能烧芯片；② 搞混「芯片 GPIO 号」和「板子丝印号」，接线时以板子丝印为准；③ 以为插上 USB 就自动有程序——其实要先烧录固件/代码。

开发环境：装好工具，给小工人装上「Python 口音」

时长 ≈ 2h · 入门阶段
🔌
心智图像：新买的板子出厂是「白纸」。我们要先往里刷一个 MicroPython 解释器（像给小工人请个翻译，让他说 Python），然后用 Thonny（一个带 Python 控制台的小工具）通过 USB 跟他对话、把程序喂进去。

选 MicroPython 的理由很朴素：你本来就懂 Python，几乎零语法转换成本。等你建立直觉后，专家阶段再学官方 C 工具链会非常快。

四步跑通

  • 第 1 步 · 装 Thonny：去 thonny.org 下载，一路下一步。它是个图形化 Python IDE，自带串口连接和文件管理。
  • 第 2 步 · 烧录 MicroPython 固件：板子插 USB → Thonny 菜单「运行 → 配置解释器」→ 选 MicroPython (ESP32) → 选对串口 → 点「安装/更新固件」，按提示把板子「BOOT 键」按住再点确定，烧完松开。
  • 第 3 步 · 连上 REPL：配置好后底部出现 >>> 提示符，这就是芯片的「实时控制台」。直接敲 print("hello esp32") 回车，他会立刻回你。
  • 第 4 步 · 跑第一个文件：在 Thonny 写代码，按 F5 就发到板子运行。想上电自动跑，把文件存成 main.py。
找不到串口 / 连接失败？Mac 一般是 /dev/cu.usbserial-*，Windows 是「设备管理器」里的 COMx。若死活连不上，多半是没装 USB 驱动（CH340 或 CP210x），去搜你板子型号 + 驱动即可。
🛠️排错卡 · 开发环境
症状

Thonny 里选不到串口，或提示 Device not found / 连不上；烧录固件一直失败。

原因

① 没装 USB 转串口驱动（CH340 / CP210x）；② 烧录时没按住 BOOT 键；③ 用的是「只充电不传数据」的线；④ 串口被别的软件占用。

解法

装对应驱动；换一根能传数据的线；烧录时按住 BOOT 再点下载、烧完松开；关掉占用串口的其它程序；Mac 在端口里找 /dev/cu.usbserial-*。

>>> import machine
>>> print(machine.ESP32_HEAP_SIZE if hasattr(machine,'ESP32_HEAP_SIZE') else 'MicroPython OK')
MicroPython OK
>>> # 看到这条，说明翻译官已上岗 🎉

面包板与电路直觉：电流像水流

时长 ≈ 2h · 入门阶段
🌊
心智图像：把导线想成水管，电压想成水压，电流想成水流，电阻想成收口的水管。电流只会从「高水压」流向「低水压」。LED 像个见水就亮的小水轮——但水太大(电流过大)会冲坏它，所以要在前面串一个限流电阻当「收口水管」。

认识面包板

面包板是带金属触点的插孔板，不用焊接就能搭电路。记住它的内部连通规律：同一行（横向 5 孔）是通的，中间凹槽上下两半各自纵向（竖向）相通。所以跨凹槽的信号要用电线引过去。

认识元件

  • 电阻：身上有彩色环表示阻值。LED 串联 220Ω~330Ω 限流最常用。
  • LED：有方向！长脚 = 正极(阳极)，短脚 = 负极(阴极)。反接不亮也不坏。
  • 导线：公对公杜邦线，用来在面包板和板子间连线。

一个最小的「灯电路」

交互 · 电流像水流点「闭合/断开」
为什么一定要串联电阻？ESP32 的 GPIO 高电平约 3.3V，而 LED 导通后「压降」只有约 2V，若不限制电流，瞬间电流可能上百 mA，会烧毁 LED 甚至 GPIO 引脚。220Ω 电阻把电流限制在 ~6mA，安全又够亮。

数字世界 · GPIO 输出：让 LED 闪起来（Blink）

时长 ≈ 1.5h · 入门阶段
💡
心智图像：GPIO 引脚就像墙上的电灯开关。你写 pin.value(1) 就是「合上开关 → 通电 → 灯亮」；写 pin.value(0) 就是「拉开开关 → 断电 → 灯灭」。芯片在这一高一低之间反复横跳，灯就闪了。

「Blink」是硬件世界的「Hello World」。它教会你三件事：怎么创建一个引脚对象、怎么设成输出、怎么用循环 + 延时制造节奏。

from machine import Pin
import time

led = Pin(2, Pin.OUT)   # GPIO2 接板载蓝灯(高电平亮)

while True:
    led.value(1)        # 开灯
    time.sleep(0.5)     # 亮 0.5 秒
    led.value(0)        # 关灯
    time.sleep(0.5)     # 灭 0.5 秒
接线（若用外接 LED）LED 长脚 → 220Ω 电阻 → GPIO（如 D4）；LED 短脚 → GND。若用板载 LED（多数板在 GPIO2），什么都不用接，直接跑代码就能看到闪。
交互 · Blink 时序拖动滑块改快慢
🛠️排错卡 · 灯不亮 / 不闪
症状

程序在跑，但 LED 不亮，或一直亮着不闪。

原因

① 外接 LED 正负极插反（长脚=阳极应接 GPIO，短脚接 GND）；② 忘记串限流电阻或阻值太大；③ 代码里的 GPIO 号和板子丝印对不上；④ 外接却接了非板载 LED 脚。

解法

先用板载蓝灯（GPIO2）跑通，排除接线问题；外接时确认长脚接信号脚、短脚经电阻到 GND；打印引脚号、对照板子丝印核对；用试灯确认 GPIO 有输出。

注意：time.sleep() 期间芯片「啥也不干、干等」，这叫阻塞。入门阶段无所谓，中级我们会学不阻塞的写法。

输入与交互：按钮、上拉/下拉电阻、读取开关

时长 ≈ 2h · 入门阶段
🔘
心智图像：按钮就是个临时接通的水龙头。没按时，引脚「悬空」像没系绳的风筝，读到的值乱跳；按下时把它接通到 GND（或 VCC），值才确定。上拉/下拉电阻就是给风筝系一根绳，让它平时有个确定的状态。

ESP32 的引脚可以设成 Pin.IN 来读外部电平。问题是：按钮没接时，引脚电平不确定（会受干扰乱跳）。解决办法是启用内部上拉电阻 Pin.PULL_UP——让引脚平时被「轻轻拉」到高电平，按下按钮才接地变低。

btn = Pin(4, Pin.IN, Pin.PULL_UP)  # GPIO4, 输入, 启用内部上拉

while True:
    if btn.value() == 0:   # 被拉低 = 按钮按下
        print("按键被按下!")
        time.sleep(0.2)      # 简单防抖
交互 · 上拉 / 下拉电阻切换模式 + 按按钮
🛠️排错卡 · 按钮读不到 / 乱跳
症状

读按钮永远返回 1，按了也没反应；或数值时不时乱跳。

原因

① 没启用内部上拉，引脚浮空受干扰；② 按钮接法错（应一端接 GPIO、一端接 GND）；③ 没防抖，机械弹跳被误判成多次。

解法

建 Pin 时加 Pin.PULL_UP；逻辑上「按下 = 读到 0」；加 0.2s 防抖或只在下降沿触发一次。参考上面的「浮空」演示就能看到差别。

永远别让引脚「浮空」去读关键状态点开上面的「无电阻(浮空)」再按按钮，你会发现读到的 0/1 是随机的。真实项目里读按钮/开关，务必启用上拉或下拉。

串口监视器：芯片怎么「说话」

时长 ≈ 1h · 入门阶段
📟
心智图像：串口是芯片给你发的短信通道。你写的 print() 不会显示在芯片上（它没屏幕），而是通过 USB 线「嗖」地发到电脑，显示在 Thonny 底部。它是你看透芯片在想什么的唯一窗户。

调试硬件 90% 靠串口：程序卡哪了？变量现在是多少？按钮到底读到没？把这些 print 出来一看便知。

  • 波特率：双方约定的「说话速度」，常用 115200。两边不一致就会显示乱码。
  • 在 Thonny：底部「Shell」区域就是串口监视器，自动按 115200 打开。
  • 小技巧：出问题时，先在代码最前面 print("boot ok")，确认程序确实跑起来了，再逐步往下加打印。
import time
for i in range(5):
    print("计数:", i)   # 这行会出现在电脑串口监视器
    time.sleep(1)
进阶：把报错也打印出来用 try / except Exception as e: print("出错:", e) 包裹可疑代码，硬件世界"看不见"，串口日志就是你的眼睛。
🛠️排错卡 · 串口乱码 / 无输出
症状

串口窗口一片乱码，或程序明明在跑却完全没输出。

原因

① 波特率两边不一致（常用 115200）；② 没真正连上 REPL（解释器没选对）；③ 程序在 print 之前就崩了。

解法

确认 Thonny 解释器设为 MicroPython (ESP32) 且端口正确；在代码最前面先 print("boot ok") 确认启动；核对波特率一致。

第一个完整项目：按钮控制灯 + 串口日志

时长 ≈ 1.5h · 入门阶段
🏗️
心智图像：前面学的都是单词，现在是第一次造句——把「读按钮」和「控灯」接在一起：按钮一按，灯就反转。串口则像黑匣子，记录每一次状态变化。

接线：按钮一端接 GPIO4，另一端接 GND（依赖内部上拉）；LED（或板载蓝灯 GPIO2）由程序控制。下面的代码实现了「按一下，灯在亮/灭之间切换」。

from machine import Pin
import time

led = Pin(2, Pin.OUT)
btn = Pin(4, Pin.IN, Pin.PULL_UP)

state = 0          # 灯当前状态 0=灭 1=亮
last = 1          # 上一次读到的按钮值(上拉:未按=1)
led.value(state)
print("系统就绪，按按钮切换灯")

while True:
    v = btn.value()
    if last == 1 and v == 0:      # 检测到「按下沿」
        state = 1 - state          # 翻转
        led.value(state)
        print("灯现在:", "亮" if state else "灭")
        time.sleep(0.2)            # 防抖
    last = v
防抖（debounce）是什么？机械按钮按下/松开瞬间会快速弹跳几次，可能被误读成「按了好几下」。上面用 0.2 秒延时 + 「只在按下沿触发一次」来过滤。后面中级阶段会学更优雅的「中断+定时器」方案。
🛠️排错卡 · 按钮控制灯不切换
症状

按一下按钮，灯不反转，或疯狂连续翻转。

原因

① 在循环里每帧都翻转，没做「沿检测」；② 没防抖，一次按下被当成多次；③ 状态变量作用域错了，每次循环都被重置。

解法

只在「上次=1 且 本次=0」的下降沿翻转一次；加 0.2s 防抖；用全局 state 变量保存灯的当前状态。
