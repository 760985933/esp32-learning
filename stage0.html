模块一 · 基础电子（模电 / 数电直觉）

A1

电压 / 电流 / 电阻：把电想成「水」

时长 ≈ 1h · 预备
🌊
心智图像：电压 = 水压，电流 = 水流，导线 = 水管，电阻 = 收口的水管。水压越大、管子越粗（电阻越小），流过的水就越多。公式 V = I × R 就是「水压 = 水流 × 水阻」——别背，先记住这张图。

电压 V

单位伏特(V)。电池/USB 给的「推力」。ESP32 用 3.3V 逻辑电平。

电流 I

单位安培(A)。实际流动的「水量」。GPIO 一般最多 ~40mA。

电阻 R

单位欧姆(Ω)。对电流的「阻挡」。串联限流、分压都靠它。

交互 · 电流像水流点按钮看水(电流)流动
记住一个安全数字ESP32 的 GPIO 是 3.3V 电平，绝不能直接接 5V 信号（会烧）；单个引脚电流别超 ~40mA，整板别超 ~200mA。这些数字后面会反复出现。
A2

电阻与分压器：用两个电阻「切」出任意电压

时长 ≈ 1h · 预备
🔪
心智图像：两个电阻串在一起，像两段不同粗细的水管。中间抽头处的「水压」（电压）取决于两段的比例。这就是分压器——电位器、旋钮、模拟量读取全靠它。

公式：Vout = Vin × R2 / (R1 + R2)。拖动下面滑块改变比例，看中间抽头电压如何从 0 平滑变到 5V——这正是 ADC（中级课 1）读到的东西。

交互 · 分压器拖滑块改 R1/R2 比例
电阻上那串色环是啥？色环是电阻的「身份证」：棕红橙黄绿蓝紫灰白黑对应数字 0–9，前两位是有效数字、第三位是倍率。新手不用硬记，用万用表或看包装标签即可。常用限流电阻：220Ω（LED）、10kΩ（上拉）。
A3

电容：能「存电」的小水箱

时长 ≈ 1.5h · 预备
🪣
心智图像：电容像一个小水箱——充电时往里灌水，放电时放出来。灌/放的速度由「水箱大小(C) × 水管粗细(R)」决定，叫时间常数 τ = R·C。所以它天生用来：滤波（ smoothing ）、定时（延时）、储能。

点按钮切换充电/放电，看电压沿指数曲线爬升或跌落——这就是所有「延时」「RC 振荡」「电源去耦」背后的数学直觉。

交互 · 电容充放电点按钮切换充/放电
电容的两个「坑」① 极性：电解电容（大个、圆柱）有正负极，接反会炸/漏液；陶瓷电容（小黄片）无极性随便接。② 单位：常用 μF（微法）和 pF（皮法），1F = 1,000,000μF，别看错数量级。
A4

电源与稳压：给芯片一口「稳」的电

时长 ≈ 1.5h · 预备
🔋
心智图像：芯片像一台精密仪器，只接受稳定的 3.3V。可电池电压会随电量掉（4.2V→3.0V），墙上电源又可能是 5V/9V/12V。于是我们要一个稳压器当「稳压水龙头」：不管进来的水压怎么晃，出来的始终是稳稳的 3.3V。

LDO 线性稳压

如 AMS1117-3.3。简单便宜、纹波小，但「压差×电流」都变成热量。适合小电流。

开关稳压(DC‑DC)

如 MP2307。效率高（90%+），大电流不烫，但电路稍复杂、有纹波。电池设备首选。

去耦电容

每个芯片电源脚旁放 0.1μF 陶瓷电容，像「小水箱」吸收瞬间波动，芯片才不抽风。

ESP32 开发板已经帮你稳压了你买的 Dev Module 板载了 USB→3.3V 稳压，插 USB 就能跑。但自己做项目/用电池时，必须自己加稳压（见中级课 8 深度睡眠接线图）。
A5

三极管 / MOSFET：用小信号「遥控」大电流

时长 ≈ 1.5h · 预备
🎚️
心智图像：三极管/MOSFET 像一个由小水流控制的水闸。基极/栅极只来一点点电流（信号），就能决定「大水流」（主回路）通不通、通多大。这就是「用小控制大」——LED 调光、电机驱动、继电器全靠它。

点按钮给基极通电，看集电极那一路大电流导通、LED 亮起。GPIO 引脚「力气小」，正是靠它才能驱动灯、电机、蜂鸣器。

交互 · 三极管开关点按钮给基极通电
三极管 vs MOSFET 怎么选？小信号/小电流用 三极管(BJT) 简单；驱动电机、大电流、省电用 MOSFET（电压控制、几乎不耗栅极电流）。ESP32 的 GPIO 是 3.3V，选「逻辑电平 MOSFET」（如 IRLZ44N 不够，需 AO3400/IRF3708 类）才能完全导通。
A6

数字逻辑基础：0 和 1 怎么「算」

时长 ≈ 1.5h · 预备
🔢
心智图像：数字世界只有两种状态：0 = 关/低电平(≈0V)，1 = 开/高电平(≈3.3V)。芯片内部亿万个「开关」按规则组合，就构成了「与/或/非」这些逻辑门——这正是你写 if (a and b) 的硬件根基。

三个最基础的逻辑门（输入 A、B，输出 Y）：

与 AND

Y = A 且 B。两者都 1 才 1。
0·0=0 0·1=0 1·1=1

或 OR

Y = A 或 B。任一为 1 即 1。
0+0=0 0+1=1 1+1=1

非 NOT

Y = 取反。0→1，1→0。
¬0=1 ¬1=0

为什么这跟你有关？GPIO 输入读到的就是 0/1；I2C、SPI、串口都是「按时钟把 0/1 一位位传」；中断是「某个脚从 0 变 1 就打断」。懂了 0/1，所有通信协议都只是「约定好的 0/1 顺序」。
数电模块从这里开始上面只是「开胃」。真正的数字电路（数电）要回答两件大事：① 怎么表示和处理数字（数制、布尔代数、组合逻辑）；② 怎么记住和按节奏处理数字（触发器、计数器、时钟、状态机）。下面 6 节把这两件事讲透，并配可交互演示——它们是理解「ESP32 为什么能跑程序、协议为什么不会乱」的底层基础。
A7

数制与码制：芯片只认 0/1，我们帮它「翻译」

时长 ≈ 1h · 预备 · 数电
🔟
心智图像：电路只有「高/低」两种电平，所以芯片内部全是二进制(0/1)。但让人直接写一串 0101 太反人类，于是我们用十六进制把每 4 位压成 1 个字符，用 BCD 让十进制数字直接对应 4 位（方便数码管显示），用 格雷码 让相邻数值只差 1 位（防止编码器读数跳变出错）。

点下面的格子切换 0/1，或拖滑块设十进制值，实时看它在四种表示法下的样子——这正是你以后读寄存器、看通信报文时天天打交道的东西。

交互 · 数制转换器点格子或拖滑块
为什么 0x 开头？代码里 0x1A 表示十六进制 1A = 二进制 0001 1010 = 十进制 26。ESP32 的地址、颜色值、寄存器都常用十六进制写，一眼看位数比十进制清楚。
A8

逻辑代数与卡诺图：用「圈格子」化简电路

时长 ≈ 1.5h · 预备 · 数电
🧮
心智图像：逻辑代数就是把「与或非」当加减法来算。核心武器是德摩根定律（「与非」可变成「或非」、反之），用来把电路改成只用一种门。再复杂，也能用卡诺图——把真值表画成方格，把相邻的 1 圈成矩形，矩形越大项越少，电路就越省。

基本运算

与·或·非 + 异或 XOR（不同为 1）。A⊕B 常用于奇偶校验、加法。

德摩根定律

¬(A·B)=¬A+¬B
¬(A+B)=¬A·¬B。门电路互转的钥匙。

卡诺图化简

圈 2/4/8 个相邻的 1 → 消去互补变量，得到最简与或式。

点「示例」载入「多数表决」（3 个输入里 ≥2 个为 1 就输出 1），看卡诺图怎么圈出 F = AB + BC + AC；也可以自己点格子搭真值表。

交互 · 卡诺图点格子设 1 / 看化简
为什么数电对你重要？ESP32 里 SPI/I2C/定时器的控制逻辑，本质都是组合+时序电路。你写 if (x & 0x80) 时，芯片内部正在做一位位的与/或/移位。懂这些，看寄存器手册不再一头雾水。
A9

组合逻辑电路：不记忆的「即时计算」

时长 ≈ 1.5h · 预备 · 数电
⚡
心智图像：组合逻辑像一台纯函数计算器——输出只取决于「此刻」的输入，不记得过去（没有时钟、没有记忆）。几个最常用的积木：

译码器（如 3-8）

3 根地址线 → 8 根里选 1 根拉高。用途：扩展 IO、点亮数码管某一位、片选。

多路选择器 MUX

像「单刀多掷开关」：用选择线从多路输入里挑一路输出。软件里的 switch 的硬件版。

加法器

半加器(无进位)/全加器(带进位) 串起来就是 CPU 里的 ALU 做加减法。

模块输入输出你会在哪见到
3-8 译码器 (74HC138)3 位地址 + 使能8 路中 1 路有效用 3 个 GPIO 控制 8 个 LED/继电器
2-1 MUX2 路数据 + 1 选择选中那一路传感器通道切换
4 位加法器两个 4 位数 + 进位和 + 进位计数、测距、PWM 计算
比较器两个 N 位数大于/等于/小于温控阈值判断
和 ESP32 的关系这些芯片现在大多「集成进 MCU 内部」了，但理解它们，你就懂了为什么 ESP32 能用几个引脚「扫」几十个按键、驱动多位数码管——背后就是译码/扫描这类组合逻辑。
A10

时序逻辑与触发器：让电路「记得」和「按拍走」

时长 ≈ 1.5h · 预备 · 数电
⏱️
心智图像：组合逻辑不记事，时序逻辑会记事。核心元件是触发器(Flip-Flop)——一个 1 位记忆单元，在时钟「滴答」的瞬间把输入锁存下来。把多个触发器并排就是寄存器，串起来就是计数器，计数器再分频就得到更慢的时钟。

下图的 D 触发器只在 CLK 上升沿采样 D。注意中间那几根「毛刺」——D 在时钟之外乱跳，Q 根本不理它，只在上升沿「咔」地抓一下。这正是「边沿触发」的精髓，也是硬件去抖的基础。

交互 · D 触发器时序看 Q 如何只在上升沿更新

触发器 D / T / JK

D：存 1 位；T：每拍翻转（做分频）；JK：功能最全。ESP32 内部成千上万。

计数器 = 分频器

n 级二进制计数器把时钟 ÷2ⁿ。进 40MHz 出 40/2ⁿ MHz——PWM、定时全靠它。

寄存器

8/16/32 个 D 触发器并排 = 一个字节/字的暂存。你写的变量最终落在这样的寄存器里。

时钟是数电的「心跳」没有时钟，时序电路就是一团乱麻。ESP32 的主频（80/160/240MHz）就是这块「心跳」——所有时序逻辑都跟着它走。下一节会看到，电平不对时这颗心跳也会出问题。
A11

逻辑家族与电平兼容：3.3V 碰到 5V 会怎样

时长 ≈ 1h · 预备 · 数电
🔌
心智图像：同一句「高电平」，不同家族理解不一样。老器件多 5V(TTL/CMOS)，ESP32 是 3.3V。就像两个人，一个说「响」指 5 分贝、一个指 3 分贝——直接连可能听不懂甚至吵坏。于是需要电平转换当翻译。

点下面四个按钮，看 3.3V 和 5V 互连的四种情形。重点记两条铁律：5V 输出绝不能直接进 3.3V 输入（会烧）；3.3V 输出驱动 5V CMOS 可能「读不准」。

交互 · 电平兼容性点按钮切换连接方案

输入阈值 VIH/VIL

高于 VIH 算 1，低于 VIL 算 0。中间是「不确定区」——这正是施密特触发要解决的。

施密特触发

上升/下降用不同阈值(迟滞)，把缓慢或带噪的信号「整形」成干净方波。按键、传感器常用。

电平转换方案

单向：电阻分压(降压)/晶体管(升压)。双向：MOSFET 或 TXS0108 专用芯片。

实战红线ESP32 几乎所有 GPIO 都不耐 5V。接 Arduino 老模块、5V 传感器时，要么选「3.3V 兼容」的，要么加电平转换。唯一例外是部分 ADC 脚可容忍更高电压（看手册），但稳妥起见一律转换。
A12

去抖与状态机：让「乱按的键」变成「确定的行为」

时长 ≈ 1.5h · 预备 · 数电
🎛️
心智图像：机械按键按下/松开时金属触点会「弹几下」，产生一串真假难辨的抖动。软件里就是「误触发多次」。去抖就是滤掉这串抖动；而状态机(FSM)是把「一连串输入 → 该干嘛」写成一张清晰的「状态转移图」，避免 if/else 嵌套成乱麻。

左边演示：同一个按键，原始信号抖了 5 下，软件若直接数会当成「按了 5 次」；去抖后只判定 1 次有效按下。右边是一个最小状态机——「智能灯」，每按一下在 关→微亮→全亮 间循环。

交互 · 按键去抖点按钮重放一次按下/松开
交互 · 状态机 FSM（摩尔机）点按钮触发状态转移

软件去抖

检测到变化后延时 ~20ms 再读一次，稳定才确认。ESP32 可用定时器/中断实现。

硬件去抖

RC 滤波 + 施密特触发，或 D 触发器链（见 A10）。按键旁常放 0.1μF 电容。

摩尔 vs 米利

摩尔：输出只由状态定（本例）；米利：输出还看当前输入。协议解析多用米利。

数电全貌小结从「0/1 怎么表示」(A7) → 「怎么算」(A8 布尔/卡诺图、A9 组合逻辑) → 「怎么记和按拍走」(A10 触发器/计数器、A11 电平) → 「怎么稳住和编排行为」(A12 去抖/状态机)，这就是数字电路的完整骨架。带着它回看四阶段，你会明白：ESP32 跑的每行代码，最终都落在这套 0/1 的时序世界里。

模块二 · PCB 设计（从面包板到可量产）

B1

为什么要从面包板走到 PCB

时长 ≈ 0.5h · 预备
🧱
心智图像：面包板是「草稿纸」——跳线一碰就松、体积大、没法带出门。PCB 是「定稿的印刷品」：所有连线蚀刻在板子里，元件焊死，稳定、小巧、可量产。做完原型想做成「能送人的产品」，必然要走这一步。

面包板阶段

验证想法、快速改线。对应前面四阶段的所有实验。

洞洞板/万用板

手工焊接过渡，适合一次性小作品。

PCB 打样

画好文件→发给打样厂(嘉立创等)→几天收到成品板。

B2

工具选型：KiCad / Altium / Cadence

时长 ≈ 0.5h · 预备

KiCad ⭐ 推荐

免费开源、跨平台、功能完整（原理图→布局→布线→3D 预览→Gerber）。社区活跃、教程多。新手无脑选它。

Altium Designer

商业软件，界面强、企业常用，但授权贵（年费上万）。找工作加分，个人学习不划算。

Cadence / Allegro

企业级、超复杂板（手机/服务器）专用。学习曲线陡，初学者一般用不到。

结论个人学习/创业/开源项目，KiCad 一条路走到底。下面所有步骤都按 KiCad 讲，但你理解的概念（封装、网络、DRC）在任意软件里通用。
B3

原理图设计（Schematic）：先画「谁连谁」

时长 ≈ 1.5h · 预备
✏️
心智图像：原理图就是「接线图的正式版」——用标准符号画元件，用网络标签(Net)标明哪根线连哪根，不用真的画一根根线。相同 Net 名字 = 电气连通（相当于面包板上同一行）。

KiCad 步骤：① 放元件（ESP32 模块、电阻、电容、连接器）；② 给每个元件指定封装(Footprint)（它将来在板子上的真实焊盘样子）；③ 连线 / 打 Net 标签；④ 跑电气规则检查(ERC)。

原理图 ≠ 板子样子原理图只表达「电气连接关系」，不表达元件摆哪里。把位置和走线留给下一步「布局布线」。
B4

封装与布局（Layout）：把元件摆上板子

时长 ≈ 1.5h · 预备
📐
心智图像：布局就是「在板子画布上摆家具」——把元件按信号流向、散热、接口方便度摆好。布局好坏直接决定后面好不好走线、板子稳不稳、好不好用。
  • 接口靠边：USB、排针、按键放板子边缘，方便插拔按压。
  • 电源/晶振近一点：稳压芯片、晶振尽量靠近用它们的 IC，线短噪声小。
  • 发热元件留空：稳压器、驱动芯片周围留散热空间，别紧贴塑料壳。
  • 天线净空：ESP32 天线下方/周围不要走线、不要铺铜、不要放金属，否则 Wi‑Fi 信号废一半。
B5

布线（Routing）：把连线「画」进板子

时长 ≈ 1.5h · 预备
🧵
心智图像：布线就是「在板子上画铜箔走线」把原理图里同 Net 的脚连起来。双面板的底层(Bottom)和顶层(Top)都能走线，中间夹一层「地平面」当公共地，用过孔(Via)打通两层。

信号线

细一点(0.2–0.3mm)即可，尽量短、少拐直角(用 45°)。

电源线

要粗(0.5–1mm+)，载流才够、压降才小。

地平面

整层铺铜当 GND，信号回流路径短、抗干扰强。

新手布线三不要① 不要天线下方/周围走线铺铜；② 不要电源线和信号线长距离平行（会串扰）；③ 不要线宽太细载不动电流（烧断/过热）。
B6

DRC 与 Gerber：检查 + 出片给工厂

时长 ≈ 1h · 预备
📤
心智图像：DRC(设计规则检查) 是「拼写检查」——自动找出线间距不够、悬空脚、短路等错误。Gerber 是「印刷底片」——工厂拿它直接在板上蚀刻铜箔。你画完板子，导出 Gerber 压缩包发给打样厂即可。
  • 跑 DRC：设好最小线宽/线距（如 6mil），确认 0 错误再出片。
  • 导出 Gerber：KiCad「绘图」选各层（Top/Bottom/Copper、Silk 丝印、Mask 阻焊、Edge 板框），含 Drill 钻孔文件。
  • 打包下单：嘉立创/捷配等，传 zip，选板材/颜色/数量，几十元起、2–3 天到手。
第一次打样建议板子做小一点（10×10cm 内常免费/极便宜）、数量 5–10 片，先用洞洞板验证过电路再投板，避免「板子到了发现原理图错了」。
B7

焊接与贴片：把元件「焊死」在板上

时长 ≈ 1h · 预备
🔥
心智图像：焊锡像「导电的胶水」——加热熔化后裹住引脚和焊盘，冷却就把两者牢牢连上并导通。手焊用烙铁+锡线；量产用钢网+锡膏+回流焊炉。

直插元件

电阻/排针，脚穿过板子，背面焊接。新手友好。

贴片 SMD

0805 电阻电容可手焊；QFP 芯片用「拖焊」；BGA 必须回流焊。

回流焊

钢网刮锡膏→贴元件→加热台/烤箱熔化。做多片才值。

焊接安全烙铁 350°C 左右，先加热焊盘再送锡，避免虚焊；ESP32 模块尽量买「已焊好排针」的 Dev Module，别自己焊 QFP。
B8

PCB 常见坑（提前避雷）

时长 ≈ 0.5h · 预备
🛠️排错卡 · PCB 设计常见故障
症状

板子回来不通电 / 芯片发烫 / Wi‑Fi 极弱 / 偶尔复位。

原因

① 线宽太细载流不足烧断；② 电源脚没放去耦电容导致芯片抽风；③ 天线下方铺铜/走线屏蔽信号；④ 封装画错（1:1 镜像、焊盘间距不对）元件焊不上；⑤ GND 没连成整片导致回流差。

解法

电源/地加粗(≥0.5mm)；每片 IC 电源脚旁放 0.1μF 陶瓷电容；天线区净空；打样前用 ERC+DRC 清零错误并双击核对封装；整板铺地铜并保持低阻抗回路。
